## 1. 项目背景与目的

### 1.1 背景
Higress 作为云原生 API 网关，具备强大的插件化和 AI 集成能力。然而，随着功能日益增加（如 AI 路由、插件配置、消费者管理等），用户在 Console 上的操作路径变长，且部分高级配置具有一定的学习门槛。

### 1.2 目的
本项目旨在为 Higress Console 引入一个**智能运维助手 (AI Agent)**，通过自然语言交互实现以下目标：
- **降低操作门槛**：用户无需记忆繁琐的配置路径，通过对话即可完成资源查询与创建。
- **提升运维效率**：支持批量查询与快速变更。
- **智能化知识库 (RAG)**：集成 Higress 官方文档与常见问题，实时解答用户配置疑问。
- **安全可控的写操作**：引入“人类在环” (Human-in-the-loop) 机制，对所有修改操作进行二次确认，防止误操作。

## 2. 方案选型与技术架构

### 2.1 技术选型
| 模块 | 选型 | 理由 |
| :--- | :--- | :--- |
| **Agent 框架** | **LangGraph (Python)** | 支持 `StateGraph` 和 `interrupt/resume` 机制，是实现“写操作确认”的最佳方案。 |
| **后端框架** | **FastAPI** | 轻量级、高性能，原生支持异步处理与 SSE 流式输出。 |
| **通信协议** | **SSE (Server-Sent Events)** | 相比 WebSocket 更轻量，适合 AI 文本流式生成，且对负载均衡更友好。 |
| **向量数据库** | **ChromaDB** | 嵌入式向量库，部署简单，适合存储本地 RAG 知识库。 |
| **前端框架** | **React + ice.js + Ant Design** | 兼容项目现有技术栈，UI 开发效率高。 |
| **生态扩展** | **MCP (Model Context Protocol)** | 支持外部 Agent（如 Claude/Cursor）直接调用 Higress 工具。 |

### 2.2 核心架构设计
- **工具化 (Tool-driven)**：将所有网关操作封装为标准 Python 纯函数，自动适配 LangChain 和 MCP 协议。
- **无状态设计**：Agent 不存储用户凭证，由前端通过 `Authorization` Header 透传 Token，确保安全性。
- **流式响应**：采用 SSE 协议实时推送 AI 思考过程、工具调用进度及最终总结。

## 3. 需求拆分 (WBS)

根据功能优先级和开发逻辑，需求拆分为以下核心模块：

### 3.1 后端 Agent 服务 (Python)
1.  **多 LLM 接入层**：对接通义千问 (Qwen)、OpenAI、DeepSeek 等主流大模型。
2.  **核心工具箱 (Core Tools)**：
    - **查询类**：查询 AI 提供者、路由、消费者、域名、后端服务等列表。
    - **操作类**：创建/更新/删除上述资源（需配合确认机制）。
3.  **智能决策引擎 (LangGraph)**：
    - 实现 ReAct 循环：意图识别 -> 工具调用 -> 结果总结。
    - **写操作拦截器**：自动识别 `create/update/delete` 操作并触发暂停。
4.  **RAG 知识库模块**：
    - 自动化索引：将 Markdown 文档转化为向量存入 ChromaDB。
    - 检索增强：AI 优先检索本地知识库回答问题。
5.  **MCP 适配器**：对外暴露标准 MCP 接口。

### 3.2 前端交互组件 (React)
1.  **悬浮聊天容器**：常驻右下角的聊天入口，支持窗口最小化/清空。
2.  **流式渲染引擎**：解析 SSE 数据流，支持打字机效果显示文字和工具状态。
3.  **三级确认卡片 (Confirm Card)**：
    - **低风险 (Summary)**：展示创建摘要。
    - **中风险 (Diff)**：展示更新前后的字段差异。
    - **高风险 (Name Input)**：删除操作需手动输入名称确认。
4.  **消息气泡扩展**：支持 Markdown 渲染（加粗、代码块、换行）和工具执行状态展示。

### 3.3 系统集成与运维
1.  **前后端代理**：配置 ice.js 与 Nginx 转发 `/agent/*` 请求。
2.  **Docker 化部署**：编写 Dockerfile 与 Docker Compose 配置文件，支持一键启动。

## 4. 写操作安全规范 (Safety First)

为确保系统安全，写操作必须遵循以下流程：
1. **意图捕捉**：AI 识别出用户想执行“写”操作。
2. **逻辑暂停**：LangGraph 进入 `interrupt` 状态，后端挂起任务。
3. **前端确认**：前端渲染 `ConfirmCard`，展示具体的参数和风险等级。
4. **人工决策**：
   - 若用户点击 [确认]，前端请求 `/chat/confirm`，后端继续执行工具调用。
   - 若用户点击 [取消]，后端丢弃该任务，AI 回复“操作已取消”。
5. **脱敏处理**：在确认卡片中，API Key、Token 等敏感信息必须自动掩码展示。
